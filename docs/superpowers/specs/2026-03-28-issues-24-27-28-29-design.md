## #24 — Scheduled Snapshot Exports

### Architecture

Background scheduler module that exports community repos as CAR files to blob storage on a configurable interval, verifies integrity, and enforces retention.

### Database Table

```sql
CREATE TABLE IF NOT EXISTS export_schedules (
    id VARCHAR(36) PRIMARY KEY,
    community_did TEXT NOT NULL REFERENCES communities(did) ON DELETE CASCADE,
    interval TEXT NOT NULL CHECK (interval IN ('daily', 'weekly', 'monthly')),
    retention_count INTEGER NOT NULL DEFAULT 5,
    enabled BOOLEAN NOT NULL DEFAULT true,
    last_export_at TIMESTAMP WITH TIME ZONE,
    last_status TEXT CHECK (last_status IN ('success', 'failed')),
    last_error TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(community_did)
);
```

### Export Flow

1. Scheduler loads all enabled schedules where `last_export_at` + interval < now
2. For each due community: `RepoEngine.exportAsCAR()` → collect all chunks into a Buffer
3. Store via `BlobStore.put()` with key format: `exports/<did>/<ISO-timestamp>.car` and mimeType `application/vnd.ipld.car`
4. Verify: `BlobStore.get()` the stored file, check first bytes are valid CAR header
5. Record metadata in `export_snapshots` table (did, timestamp, size, cid, storage key)
6. Enforce retention: list snapshots for this community, delete oldest beyond `retention_count`
7. Update `export_schedules.last_export_at` and `last_status`
8. Audit log on success and failure

### Snapshot Metadata Table

```sql
CREATE TABLE IF NOT EXISTS export_snapshots (
    id VARCHAR(36) PRIMARY KEY,
    community_did TEXT NOT NULL,
    storage_key TEXT NOT NULL,
    size_bytes INTEGER NOT NULL,
    root_cid TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX IF NOT EXISTS idx_export_snapshots_did ON export_snapshots(community_did);
```

## #27 — Configurable Per-Collection Governance

### Approach

Modify `enforceGovernance()` to read `protectedCollections` from the community's `governanceConfig`. Mandatory collections (`community.settings`, `community.role`) cannot be removed.

### Changes

**`src/governance/enforcement.ts`:**

```typescript
// Current: hardcoded PROTECTED_COLLECTIONS
// New: read from governanceConfig, fall back to hardcoded

const MANDATORY_PROTECTED = [
  'net.openfederation.community.settings',
  'net.openfederation.community.role',
];

const DEFAULT_PROTECTED = [
  ...MANDATORY_PROTECTED,
  'net.openfederation.community.member',
  'net.openfederation.community.profile',
  'net.openfederation.community.attestation',
];
```

In `enforceGovernance()`, after fetching the settings record:
```typescript
const configProtected = settings.governanceConfig?.protectedCollections;
const protectedCollections = configProtected?.length
  ? [...new Set([...MANDATORY_PROTECTED, ...configProtected.map(c =>
      c.startsWith('net.openfederation.community.') ? c : `net.openfederation.community.${c}`
    )])]
  : DEFAULT_PROTECTED;
```

**`src/api/net.openfederation.community.setGovernanceModel.ts`:**
Add validation for `protectedCollections` in `governanceConfig` — must be an array of strings, each must start with `net.openfederation.community.`.

### Files

| File | Responsibility |
|------|----------------|
| **Modify:** `src/governance/enforcement.ts` | Read protectedCollections from config |
| **Modify:** `src/api/net.openfederation.community.setGovernanceModel.ts` | Validate protectedCollections |

## #28 — Proposal Amendments

### Approach

New endpoint to amend an open proposal. Resets votes, extends expiration, preserves amendment history.

### Record Change

Proposal records gain an `amendments` array:
```json
{
  "...existing fields...",
  "amendments": [
    {
      "amendedBy": "did:plc:...",
      "previousRecord": { "...old proposed record..." },
      "amendedAt": "2026-03-28T...",
      "reason": "Updated visibility setting"
    }
  ]
}
```

### Endpoint

| Method | NSID | Auth | Description |
|--------|------|------|-------------|
| POST | `net.openfederation.community.amendProposal` | Voter role | Amend an open proposal |

**Input:** `{ communityDid, proposalRkey, proposedRecord, reason? }`

**Behavior:**
1. Verify caller has `community.governance.write` permission
2. Fetch proposal, verify status is `open`
3. Save current `proposedRecord` into `amendments` history
4. Replace `proposedRecord` with the new one
5. Reset `votesFor` and `votesAgainst` to `[]` (no auto-vote — amender must vote explicitly)
6. Extend `expiresAt` by the original TTL (from `governanceConfig.proposalTtlDays`)
7. Write updated proposal record
8. Audit log

| File | Responsibility |
|------|----------------|
| **Create:** `src/api/net.openfederation.community.amendProposal.ts` | Amendment handler |
| **Create:** `src/lexicon/net.openfederation.community.amendProposal.json` | Lexicon |
| **Modify:** `src/db/audit.ts` | Add `community.proposal.amend` action |
| **Modify:** `src/server/index.ts` | Register handler |

## #29 — Delegation / Proxy Voting

### Approach

New collection for delegation records. Vote counting in `voteOnProposal` automatically includes delegated votes.

### Delegation Collection

```
Collection: net.openfederation.community.delegation
rkey: TID
```

```json
{
  "delegatorDid": "did:plc:alice",
  "delegateDid": "did:plc:bob",
  "createdAt": "2026-03-28T..."
}
```

One delegation per member per community (enforced at application level — latest delegation wins).

### Endpoints

| Method | NSID | Auth | Description |
|--------|------|------|-------------|
| POST | `net.openfederation.community.setDelegation` | Voter role | Delegate vote to another member |
| POST | `net.openfederation.community.revokeDelegation` | Voter role | Remove delegation |
| GET | `net.openfederation.community.getDelegation` | Any member | Check delegation status |

### Vote Counting Change

In `voteOnProposal`, after recording the direct vote:
1. Query `net.openfederation.community.delegation` records where `delegateDid` = voter's DID
2. For each delegator found:
   - Check if delegator has already voted directly on THIS proposal (skip if so)
   - Add delegator's DID to the same vote array (for/against) as the delegate voted
3. Then proceed with quorum check as normal

Direct votes always override delegations.

## What's NOT Included

- Admin dashboard UI for export status (#24) — API endpoints only, UI is a future enhancement
- Email alerts on failed exports — audit log + API is sufficient for MVP
- Cross-community delegation — delegation is per-community only
- Weighted voting — all votes count equally regardless of role
